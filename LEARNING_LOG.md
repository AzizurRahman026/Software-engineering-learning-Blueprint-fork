## 2026-07-08 — Day 29 — AI: Guarding & validating LLM output (prompt-injection fencing + output allow-listing)

**What was covered:** The security half of the structured-output thread (Days 24/26 did the typed-parse + business-validation half). Two doors defended on the `SuggestAndSaveThreadTitle` slice: (1) INPUT — the untrusted first user message spliced into the title prompt is now fenced as DATA (`WrapUntrustedContent`) with a system-prompt instruction (`FenceInstruction`) telling the model never to obey fenced text, and the fence marker is defanged inside the content so it can't be forged (prompt-injection defense-in-depth, explicitly NOT a guarantee); (2) OUTPUT — the parsed title + topics are untrusted text bound for the Angular sidebar (HTML/markdown context) and logs, so they run through `SanitizeDisplayText` which allow-lists down to plain display text (strips control chars + `< > \` [ ]`, collapses whitespace, trims quotes, caps length, safe fallback), replacing the ad-hoc `Trim().Trim('"')` + manual cap. Simplicity gate: "do nothing" / inline `.Trim()` was fine while the only worry was stray quotes + length; it breaks on markup injection and prompt injection, and once a second call site (topics) wants the same treatment the ad-hoc chain is just duplicated + incomplete — so one allow-list helper earns its place. Core distinction taught: allow-list beats blocklist (`<scr<script>ipt>` survives a blocklist), and parsed ≠ safe.

**Senior/Architect lens:** Boundary + Review — sanitize vs. reject is context-dependent: sanitize+degrade for COSMETIC output (a mangled chat title is harmless, keep the feature alive), but REJECT+fail-closed for AUTHORITATIVE output (routing keys, amounts, tool args, permissions) because a silently-sanitized authoritative value is a wrong value that looks fine. Failure signal: prompt injection succeeds QUIETLY (no exception) — the observable is a "sanitizer stripped a lot / fallback fired" warning counter per provider. System fit: SendChatCommandHandler's agentic loop (tool args + tool results fed back to the model) is the bigger untrusted surface and is NOT fenced yet — added to the watch list, out of weekday scope.
